<h1> Parcial 3er Corte </h1>  

Este proyecto implementa un cpipeline de integracion continua usando lo siguiente
* Java con Maven
* Junit para los test unitarios
* Super Linter
* GitHub Actions como plataforma CI/CD

<h2>Objetivo</h2>
El objetivo es asegurar que en cada push o pull valide la calidad del codigo antes de integrarsen antes de realizar la integracion con la rama principsl
Con una cobertura del 70%

<h2> Ejecucion local con Nektos act
La herramienta Nektos nos permite ejecutar GitHub Actions de foma local
requiere de Docker

<h2>Ejecucion localmente</h2>
Act push

<h3>Diferencia entre CI y CD</h3>
El objetivo de CI es detectar errores antes, mantener el codigo testeado, mientras que CD realiza la publicaciones y despliegues
